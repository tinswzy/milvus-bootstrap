环境准备 · 远程 minikube + SSH 隧道

本地资源有限 → 把集群放到 远程主机（32G / 16core），本地 kubectl / mb 经 SSH 隧道连过去。三段式：A 远程起集群 · B 本地开隧道（常开）· C 本地跑 mb。

一次性准备远程 minikube推荐验证环境
← 返回清单

🗺 拓扑

本地笔记本
mb daemon · kubectl · helm
⇄SSH 隧道
127.0.0.1:8443
远程主机 32G/16core
minikube（docker 驱动）
apiserver 127.0.0.1:8443

两端都用 127.0.0.1:8443，所以 kubeconfig 不用改 server、不用 skip TLS（证书 SAN 自带 127.0.0.1）。固定 --apiserver-port=8443 让端口不随重启变。

窗口 A · 远程起 minikube远程 shell

在远程主机上执行（幂等：已起则跳过）
ssh user@<远程IP> 后
minikube start --driver=docker --apiserver-port=8443 --memory=24g --cpus=12
😄 minikube ... Done! kubectl is now configured
kubectl config view --flatten --minify > ~/mk-remote.yaml # 证书内嵌，给本地用
kubectl get nodes # 远程本机自检
NAME STATUS ROLES AGE VERSION minikube Ready control-plane 1m v1.3x
32G/16core 给 minikube 留足（--memory=24g --cpus=12，余量留宿主）。--apiserver-port=8443 固定端口，隧道永远连 8443。

窗口 B · 本地开 SSH 隧道本地 · 常开

这个隧道要一直开着；断了 kubectl/mb 就连不上
本地 shell（专用于隧道）
scp user@<远程IP>:~/mk-remote.yaml ~/.kube/mk-remote.yaml # 拿 config（一次）
ssh -N -L 8443:127.0.0.1:8443 user@<远程IP> -o ServerAliveInterval=30 -o ServerAliveCountMax=3
# 不返回、保持前台 = 隧道在工作；想后台加 -f
本地 127.0.0.1:8443 ⇄ 远程 127.0.0.1:8443。隧道挂了重跑这条即可（幂等）。更稳用 autossh。

窗口 C · 本地接 mb本地 · 跑命令

所有 mb/kubectl 命令都在这个 shell 跑
本地 shell（跑 mb，每条幂等）
export KUBECONFIG=~/.kube/mk-remote.yaml # 指向远程集群（本 shell 一次）
kubectl get nodes # 经隧道直连远程，应 Ready
cd milvus-bootstrap && source .venv/bin/activate
export MB_ADAPTER=k8s
mb core stop && mb core start # daemon 继承 KUBECONFIG+adapter，必须在此之后起
mb status # 确认 adapter=k8s
helm repo add bitnami https://charts.bitnami.com/bitnami && helm repo add apache https://pulsar.apache.org/charts && helm repo update
顺序要点：KUBECONFIG 和 MB_ADAPTER=k8s 必须在 mb core start 之前 export（daemon 启动时定死）。改了任一个都要 mb core stop && mb core start。

装 foundation（全新集群需要）

远程是全新 minikube → 先装 etcd + minio，milvus 才有依赖可连
本地窗口 C
mb install etcd --name etcd --apply # 命名 etcd → 服务 etcd.default.svc:2379（贴合 milvus 默认）
mb install minio --name minio --apply
kubectl get pods # 期望 etcd / minio Running
命名为 etcd/minio 让 milvus 默认 endpoint（etcd.default.svc/minio.default.svc）直接解析。装完 kubectl get svc 确认服务名；不符则步骤④用 --set etcdEndpoints=… --set storageEndpoint=… 指向实际服务。全部装在 default ns，后续步骤命令（不带 -N）原样可用。

✅ 就绪检查

  • kubectl get nodes → minikube Ready（经隧道）
  • mb status → adapter = k8s
  • kubectl get pods → etcd / minio Running
  • helm repo 有 bitnami / apache
🟢 四项都对 → 环境就绪，去 步骤 ① 开跑。远程 32G/16core，前面担心的 pulsar/switch 资源问题都不再是瓶颈。

🧹 复位 / 重来

复位
# 清单个组件：mb delete <name> --apply
# 整个集群重来（远程）：minikube delete && minikube start --apiserver-port=8443 --memory=24g --cpus=12
# 隧道重开（本地）：重跑窗口 B 那条 ssh -L
隧道/集群都可随时重来；minikube start 幂等，端口固定 8443 不变 → 隧道和 config 不用动。

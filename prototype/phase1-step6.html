步骤 ⑥ · switch-mq：kafka → pulsar（★ 核心）

不重装 milvus，在线把 WAL/MQ 从 kafka 切到 pulsar —— 这是整个工具的招牌能力。

☸️ 实测需 minikube待验收
← 返回清单
🔁 验证节奏：① 我执行一遍 → 绿灯 → ② 清理/复位（切回 kafka）→ ③ 你复跑确认顺畅。

🎯 目标 & 前置

三步护栏：装新 MQ → wal/alter → 下线旧。新 MQ(pulsar)在步骤②已就绪，milvus(kafka)在步骤④已 Healthy。

步骤 ④ milvus-dev Healthy（on kafka）步骤 ② pulsar-dev Running

🚀 前置（幂等 · 可重复跑）

从干净或脏状态都能从头跑到尾
🚀 环境（每条都幂等）
cd milvus-bootstrap && source .venv/bin/activate
export MB_ADAPTER=k8s # 真集群；改 adapter 要 mb core stop 再 start
mb core start # 幂等：已运行则跳过
mb status # 确认 adapter=k8s 再往下
前序产物：需 milvus-dev on kafka（步骤④）Healthy + pulsar-dev（步骤②）已在。幂等说明：switch-mq 可来回切（切回 kafka 即复位），重复跑安全 → 可从头到尾反复执行观察。

📋 任务细则

MilvusDriver.plan_switch_mq_steps
  1. precheck-target：确认目标 MQ(pulsar)服务就位、milvus 可连。
  2. wal-alter：在 milvus pod 内 POST :9091/management/wal/alter，body {"target_wal_name":"pulsar"}（经 adapter.exec 执行）。
  3. verify：校验 WAL 已切到 pulsar、旧 MQ 写入排空。
  4. decommission-old：确认无残留写入后下线旧 MQ。

✅ 验收命令

在 minikube 上跑
☸️ 切换 + 校验
mb switch-mq milvus-dev --to pulsar --apply
precheck-target ok wal-alter ok (POST /management/wal/alter target_wal_name=pulsar) verify ok decommission-old ok
kubectl get milvus milvus-dev -o jsonpath='{.status.status}'
Healthy
🟡 绿灯判据（期望，待真集群回填实测）：wal/alter 返回 ok；milvus 切后仍 Healthy；新写入落到 pulsar（旧 kafka 不再产生新写）。

🧹 清理 / 复位

🧹 复位（切回 kafka）
mb switch-mq milvus-dev --to kafka --apply # 切回，便于你复跑
切回 kafka 即回到步骤④状态，你可从干净起点复跑切换。

  • 新写入走 pulsar

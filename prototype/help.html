这是什么 · 架构

Milvus Admin WebUI 是一个 pip 安装即用的 Python WebUI，连接 Kubernetes，对 Milvus 及其依赖组件做官方方式（operator / helm）层面的可视化安装与配置管理。它本身尽可能无状态——只是一个管理者，每个组件都由各自的官方安装方式（operator / helm） 独立维护。

安装与启动Python module · pip
bash — milvus-admin-webui
# 1) 直接 pip 安装（纯 Python module）
$ pip install milvus-admin-webui

# 2) 连上你的 kubeconfig 启动 WebUI
$ milvus-admin-webui --kubeconfig ~/.kube/config --port 8080
✔ connected to cluster: prod-cluster
✔ discovered: milvus-operator · minio-operator · strimzi · etcd/pulsar (helm)
✔ serving UI at http://localhost:8080

四条核心原则
1

无状态 Stateless

不自带数据库、不持有集群与组件关系状态。视图实时读自 k8s 与各组件 ConfigMap，刷新即重建。唯一真相在集群里，不在本工具。

2

各管各 · 各装各的

每个组件由它自己的官方安装方式（operator / helm） 独立安装与维护 pod。各组件之间互不依赖，每个只做自己的事；删除一个不会连带影响其它。

3

关系由配置决定

Milvus 连哪个 MQ、用哪个对象存储，是各自 ConfigMap 的事。WebUI 只负责把配置改对、触发重启，不编排隐式依赖，也不越界代管别的组件。

4

即装即用 · 只做可视化与运维

一个 Python 包，pip 安装、连上 kubeconfig 即可。对底层资源做官方方式的安装/配置可视化，并提供必要的常用运维工具，仅此而已。

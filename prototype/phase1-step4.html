✅ 实测结论（2026-07-01）：milvus 2.6.3 经 operator 全 external 连 etcd+minio+kafka，.status.status=Healthy、5 个依赖条件全 True、mb 登记 Managed。命令 exit 0（apply-cr → wait-status(Healthy) → register 四步全 ok）。
⚠️ 关键坑 · minio 端点必须是单字段 host:port（driver 已修）：operator 把 storage.endpoint 拆成 minio.address+minio.port 两字段后，milvus segcore「init vector storage」会把 bucketName 拼进 endpoint（host:port/bucket）→ minio-go 报 Endpoint url cannot have fully qualified paths → CrashLoopBackOff（2.5/2.6 同病，只是先崩的组件不同）。修复：milvus.py 在外部 MinIO 时注入环境变量 MINIO_ADDRESS=<host:port>（milvus 优先读该 env 且保留冒号原样），绕开拆字段。改 driver 代码后需 mb core stop && mb core start 重载 daemon 才生效。

这是 kafka 闭环的核心一格：milvus 不装任何依赖 pod，全部 external 连上前面装好的三件套。

📋 任务细则

  1. 装 milvus：operator-cr apply Milvus CR（mq=kafka，brokerList 指向 kafka-dev），步骤 precheck-operator → apply-cr → wait-status(Healthy) → register。
  2. 就绪门：operator 把 .status.status 推到 Healthy，且 MsgStreamReady=True。
  3. 登记：Managed · stateless，spec_snapshot 落库供后续升级/切换复用。

✅ 实测结论（2026-07-01）：milvus-dev 由 v2.6.3 升到 v2.6.18，operator 滚动（新 pod ready 后旧 pod 才 Terminating），.status.status 回 Healthy，etcd/minio/kafka 全程不动（age 均早于升级）。升级 re-render 从快照重建 CR，MINIO_ADDRESS env 自动保留——这也顺带验证了 2.6.18（裸配置必崩的版本）带修复后可正常运行。

🎯 目标 & 前置

milvus 是 stateless，升级 = 改 CR components.image 后 operator 滚动；etcd/minio/kafka 全程不动。

步骤 ④ milvus-dev Healthy

🚀 前置（幂等 · 可重复跑）

从干净或脏状态都能从头跑到尾
🚀 环境（每条都幂等）
cd milvus-bootstrap && source .venv/bin/activate
export MB_ADAPTER=k8s # 真集群；改 adapter 要 mb core stop 再 start
mb core start # 幂等：已运行则跳过
mb status # 确认 adapter=k8s 再往下
前序产物：需 milvus-dev（步骤④）已在。幂等说明：mb upgrade --image 再渲染应用，重复/回旧镜像都安全 → 可从头到尾反复执行观察。

📋 任务细则

  1. 升级 milvus：mb upgrade milvus-dev --image milvusdb/milvus:v2.6.4 --apply → 重渲染 CR 的镜像、operator 滚动。
  2. 就绪门：滚动后 .status.status 回到 Healthy。
  3. 依赖不动：etcd/minio/kafka 实例无变更。

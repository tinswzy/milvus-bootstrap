🎯 目标 & 前置

真集群装 MQ。kafka 默认走 bitnami-helm、pulsar 走 apache-helm；helm 路径 render → apply → wait-ready → register。

步骤 ① 绿（渲染正确） minikube 就绪（kubectl get nodes Ready） daemon 用 k8s adapter

🚀 前置（幂等 · 可重复跑）

从干净或脏状态都能从头跑到尾
🚀 环境 + helm 仓库（每条都幂等）
cd milvus-bootstrap && source .venv/bin/activate
export MB_ADAPTER=k8s # 真集群；改 adapter 要 mb core stop 再 start
mb core start # 幂等：已运行则跳过
mb status # 确认 adapter=k8s 再往下
helm repo add bitnami https://charts.bitnami.com/bitnami # 幂等：已存在则覆盖
helm repo add apache https://pulsar.apache.org/charts
helm repo update
幂等说明：本页命令可反复跑——helm repo add/mb core start 已在则跳过；mb install --apply 即 helm upgrade --install（重复=覆盖）；mb delete 实例不在也安全。故可从头执行到尾、反复观察现象。

📋 任务细则

  1. 装 kafka：mb install kafka --name kafka-dev --apply → helm 装 bitnami/kafka（broker StatefulSet）。
  2. 装 pulsar：mb install pulsar --name pulsar-dev --apply → helm 装 apache/pulsar。
  3. 就绪等待：driver 的 wait-ready 按 app.kubernetes.io/instance 轮询到工作负载 Ready。
  4. 登记：装完登记为 Managed · authoritative，mb discover 可见。

✅ 验收命令

在 minikube 上跑
☸️ 安装 + 校验
mb install kafka --name kafka-dev --apply
mb install pulsar --name pulsar-dev --apply
kubectl get pods | grep -E 'kafka-dev|pulsar-dev'
kafka-dev-controller-0 1/1 Running pulsar-dev-zookeeper-0 1/1 Running pulsar-dev-bookie-0 1/1 Running pulsar-dev-bookie-1 1/1 Running pulsar-dev-broker-0 1/1 Running pulsar-dev-proxy-0 1/1 Running
mb status # 权威 Managed 态
instances: ['etcd', 'kafka-dev', 'minio', 'pulsar-dev']
✅ 绿灯（实测 2026-06-29）：kafka-dev / pulsar-dev broker pod Running；mb status 两者均登记为 Managed。两者都起来 → 进 步骤③。
注：mb discover 是无状态集群扫描，对已登记的 Managed 实例仍标 adoptable（discover 未 reconcile state / register 未给集群 workload 打 managed 注解）——权威 Managed 态以 mb status 为准。

🧹 清理 / 复位

我跑完即复位，让你从干净状态复跑
🧹 复位
mb delete kafka-dev --apply
mb delete pulsar-dev --apply # helm uninstall（PVC 默认保留）
kubectl delete pvc -l app.kubernetes.io/instance=kafka-dev # 彻底复位（可选）
复位后照「✅ 验收命令」再装一遍应同样 Running → 步骤可复现。

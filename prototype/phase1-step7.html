🚀 前置（幂等 · 可重复跑）

🚀 环境（每条都幂等）
cd milvus-bootstrap && source .venv/bin/activate
export MB_ADAPTER=k8s # 真集群；改 adapter 要 mb core stop 再 start
mb core start # 幂等：已运行则跳过
mb status # 确认 adapter=k8s 再往下
前序产物：需 kafka-dev + milvus-dev on kafka（步骤④）已在。幂等说明：mb upgrade --image 是再渲染应用，重复/回滚旧镜像都安全 → 可从头到尾反复执行观察。

📋 任务细则

  1. 升级 kafka：mb upgrade kafka-dev --image <newver> --apply → 重渲染 helm values 的镜像并滚动。
  2. 滚动观察：kafka broker 逐个滚动到新版本，期间 milvus 连接保持。
  3. 解耦校验：升级窗口内 milvus .status 始终 Healthy（不随 kafka 重启而 fail）。
